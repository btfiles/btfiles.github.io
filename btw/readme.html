Behind The Wall - QUTE

This is the WebGL application used in the BTW study.

To work with this sample, you should have Node (and in particular, npm and npx) installed, which you can retrieve from nodejs.org.

Running

You should first clone this repository and change directory to this folder.

Then, you set up the initial project by pulling the dependencies from npm with 

npm install

After that, you can compile and run a server with:

npm run start

For those familiar with Typescript, you do not have to run tsc to build the .js files from the .ts files; npx builds them on the fly as part of running webpack.

You can run the sample by pointing your web browser at https://localhost:8080/index.html

If you want to deploy this demo to a server, you can build this demo with:

npm run build

You will find the built web page under a directory called dist/ . You can copy everything inside it to the server. You also need to copy the assets/ folder together with the files inside the dist/ so that the 3D meshes can be found by the js. So, all you need to run the web page are:

assets
index.html
js

Changing The Code For Another Study

The following data fields inside the class Game in index.ts can be modified in case anyone wants to conduct another study using this demo.

Starting position

This array contains the starting positions of the human figure. You can replace them with other random draws.

readonly enemyOriginalX: Array<Array<Array<number>>>

Trial number

maxTrialNumber is the number of trials in each visualization condition. Right now it's 72 for each vis condition, making it 144 in total.

readonly maxTrialNumber: number

Block size

Participants will be asked to take a break after finishing breakBlockSize number of trials. 

readonly breakBlockSize: number

Uncertainty Visualization 3D meshes

If you used other 3D meshes, you might need to modify the file names in:

readonly uncertaintyVisMeshFileName: Array<string>

This demo uses gltf files. All of them were stored in a folder under assets/gltf/. The filename is the same as the folder's name. You may need to change some of the code if you plan to use other mesh formats.

URL parameters

Either of these two parameters can be left empty.

tt

The parameter 'tt' is short for trial type (or visualization type). This will determine the initial visualization type of the study. Its value is corresponding to the TrialType enum. For example, tt=0 means QDP. Without this parameter, the initial visualization type will be determined by 

private initialTrialType = TrialType.WithoutUncertainty;

h

This parameter will take in the pixel density of the display measured by the credit card question in Qualtrics, which can be used to adjust and unify the size of visual stimulus. The pixel density is measured in pixels/10cm .

An example of URL: [https://www-users.cselabs.umn.edu/~nie00035/btw/?tt=0&h=505]. Here, 505 is the pixel density of a 15" MacBook Pro.
